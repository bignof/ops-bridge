# 后续路线规划

## 近阶段

- 统一镜像命名、环境变量命名和部署模板
- 增加跨服务联调脚本，减少手工回归
- 梳理消息契约，准备共享协议层
- 明确一期验收 checklist 和发布流程

## 二期建议

- 引入批量任务模型，而不只是单条命令投递
- 支持分批更新、失败回滚和灰度控制
- 增加更完整的审计字段和操作追踪
- 完善 MySQL / SQLite 之外的部署策略和运维规范
- 给平台侧补最小可用管理界面

## 三期方向

- 多环境、多集群和多租户隔离
- 更细粒度的权限和审批流
- 可扩展执行器，不局限于 Docker Compose
- 统一事件总线、告警和观测体系

## 组织建议

如果后面团队会继续扩人，推荐按照下面的节奏推进：

1. 先稳住一期闭环和交付质量。
2. 再抽协议层和联调工具。
3. 最后再做 UI、批量编排和平台化能力。

这样能避免系统还没稳定，就过早进入复杂平台建设。
